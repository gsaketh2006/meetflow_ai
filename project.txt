# project.txt - MeetFlow AI: Notes-to-Action Assistant

---

### 1. Project Overview

**MeetFlow AI** is a sophisticated, AI-powered meeting productivity tool designed to transform raw, messy meeting notes into structured, actionable plans. By leveraging Large Language Models (LLMs), the application automatically identifies specific tasks, assigns them to team members, detects deadlines, and categorizes them by priority. It provides a seamless transition from a text transcript to a sprint-ready action plan, ensuring that no critical point is forgotten after a meeting.

**Core Idea:**
- **Automated Extraction:** No more manual note-taking for action items.
- **Structured Data:** Converts unstructured text into JSON-formatted tasks.
- **Accountability:** Clearly identifies "Who" does "What" by "When".
- **Visual Clarity:** Uses a premium "Glassmorphism" dashboard to organize tasks by priority.

---

### 2. Complete Workflow (Step-by-Step Execution)

1.  **Authentication:** The user starts by landing on the index page where they can Register or Login. Authentication is handled securely via **Supabase Auth**.
2.  **Dashboard Access:** Once logged in, the user is redirected to the Dashboard, which serves as the command center for processing notes.
3.  **File Upload:** The user uploads a meeting transcript in `.txt` format via a drag-and-drop zone or file browser.
4.  **Backend Processing:**
    *   The file is sent to the `/upload_notes` endpoint in `app.py`.
    *   The server saves the file temporarily and reads its content.
    *   The `MeetFlowAgent` (from `llm_agent.py`) is invoked to process the text.
5.  **LLM Analysis:** 
    *   The agent sends the notes to the **Groq API (Llama 3 model)** with a specialized "System Prompt".
    *   The LLM analyzes the text and returns a strictly formatted JSON object containing extracted tasks.
6.  **Confidence Calculation:**
    *   The backend runs a hybrid logic function `calculate_confidence()` to score the reliability of each extracted task based on factors like detected deadlines, assignees, and AI-provided refinement scores.
7.  **Frontend Visualization:**
    *   The processed tasks are sent back to the browser as a JSON response.
    *   `script.js` dynamically renders these tasks into three sections: **High, Medium, and Low Priority**.
    *   A "Confidence Bar" is shown for each task to indicate AI certainty.
8.  **Persistence:** The user can click "Save to Database" to permanently store the action plan in **Supabase DB**.
9.  **History Management:** Users can visit the `/history` page to view, search, filter, or delete previously generated tasks.

### 5. Function-by-Function Breakdown

#### Function: login_required (app.py)
*   **What it does:** A "decorator" that protects certain pages.
*   **Why used:** To ensure only logged-in users can see the Dashboard or History.
*   **Logic:** It checks if 'user' exists in the session. If not, it redirects to the login page.

#### Function: calculate_confidence (app.py)
*   **What it does:** Calculates a score (0.0 to 1.0) for each task.
*   **Input:** `task_obj` (The task data from AI).
*   **Output:** A float value representing confidence.
*   **Logic:** 
    1.  Starts with a base of 0.6.
    2.  Adds 0.1 if a task description exists.
    3.  Adds 0.1 if a valid deadline is found.
    4.  Adds 0.1 if a specific person (not 'Team') is assigned.
    5.  Adds 0.1 for 'High' priority.
    6.  Adds a small refinement value (0.0 to 0.1) provided by the AI itself.

#### Function: upload_notes (app.py - POST /upload_notes)
*   **What it does:** Processes the uploaded `.txt` file.
*   **Logic:** 
    1.  Receives file via POST request.
    2.  Reads the text content.
    3.  Calls `agent.process_notes(notes_text)`.
    4.  Iterates through resulting tasks and attaches confidence scores.
    5.  Returns JSON with tasks, overall confidence, and processing time.

#### Function: save_tasks (app.py - POST /save_tasks)
*   **What it does:** Saves tasks to Supabase.
*   **Logic:** Prepares a list of task objects with the current `user_id` and performs a bulk insert into the `action_plans` table.

#### Function: process_notes (llm_agent.py)
*   **What it does:** The main AI logic.
*   **Logic:** 
    1.  Defines a "System Prompt" that tells the AI to be a Senior Project Manager and output ONLY JSON.
    2.  Sends the user's notes to Groq.
    3.  Parses the JSON response.
    4.  Normalizes priority labels (e.g., making sure "high" becomes "High").

#### Function: extract_tasks (tools.py)
*   **What it does:** A keyword-based fallback for finding tasks.
*   **Logic:** Splits text into lines and looks for "need to", "must", or bullet points like "-" or "*".

#### Function: identify_deadlines (tools.py)
*   **What it does:** Finds dates in text.
*   **Logic:** Uses `dateparser.search` to find strings that look like dates and converts them to a standard YYYY-MM-DD format.

### 6. LLM Integration Details

*   **Provider:** Groq (High-performance inference).
*   **Model:** `llama-3.3-70b-versatile`.
*   **Prompt Structure:** The system prompt is the "Brain" of the extraction. It contains:
    *   **Role definition:** "Expert senior project manager".
    *   **Format constraint:** "MUST respond ONLY with a valid JSON object".
    *   **Extraction Guidelines:** Specific rules for deadlines, priorities, and confidence refinement.
*   **Processing:** The code uses `response_format={"type": "json_object"}` which is a native model feature to guarantee that the AI output is valid JSON, preventing the app from crashing on bad data.

---

### 7. Tool Functions Analysis

*   **extract_tasks():** Uses **Keyword Matching**. It is very fast but less "smart" than the LLM. It acts as a baseline to ensure nothing obvious is missed.
*   **identify_deadlines():** Uses **Date Parsing Libraries**. It is highly accurate for specific dates (e.g., "April 26") and relative dates (e.g., "next Wednesday").
*   **extract_assignees():** Uses **Named Entity Recognition (NER)** via SpaCy. It looks for "PERSON" entities in the text to identify who is being talked about.
*   **detect_priority():** Uses **Heuristic Matching**. It searches for "urgency" keywords.

### 8. Data Flow Diagram (Text Explanation)

1.  **User Input:** User uploads a transcript on the Dashboard.
2.  **API Layer:** `app.py` receives the file and extracts the text.
3.  **Agent Layer:** `llm_agent.py` sends the text + instructions to Groq.
4.  **AI Engine:** Llama 3 processes the meeting logic and identifies tasks.
5.  **Refinement:** Backend calculates confidence and organizes by priority.
6.  **UI Display:** `script.js` renders the task cards with animations.
7.  **Storage:** User saves tasks to the Supabase Postgres database.

### 11. Strengths of Implementation

*   **Premium UI:** Uses "Glassmorphism" (semi-transparent blurred backgrounds) and smooth CSS animations for a modern feel.
*   **Hybrid Scoring:** Doesn't just trust the AI blindly; it verifies the AI's work with local logic to calculate a "Confidence Score".
*   **Scalability:** Uses Supabase for the backend, making it ready to handle many users.
*   **Developer Friendly:** Robust logging and configuration validation make it easy to debug.

---

### 12. Limitations (Based on Code)

*   **File Support:** Currently only supports `.txt` files; cannot process PDFs or Audio directly.
*   **Hardcoded Keywords:** Some priority detection in `tools.py` relies on a fixed list of words.
*   **Context Limit:** Very long meeting notes (e.g., 2-hour transcripts) might hit LLM token limits (though 2048 tokens is usually plenty for most notes).
